17)База данных Oracle создает серверные процессы для обработки запросов пользовательских процессов, подключенных к экземпляру. 

ОТветы 6
1. Принцип установления соединения с сервером Oracle по сети:
   - Клиент инициирует запрос на подключение к базе данных.
   - Сетевой драйвер клиента (Oracle Net) преобразует запрос в пакеты данных, которые могут быть переданы по сети.
   - Сетевой драйвер сервера (Oracle Net) на стороне сервера принимает запрос, интерпретирует его и передает запрос экземпляру базы данных.
   - Экземпляр базы данных проверяет подлинность пользователя, авторизует его и устанавливает сессию.
   - Результат возвращается клиенту по обратному пути.

2. Назначение файлов:
   - SQLNET.ORA: Содержит параметры конфигурации сетевого программного обеспечения Oracle Net.
   - TNSNAMES.ORA: Содержит описания сетевых подключений (коннекторов) к базам данных.
   - LISTENER.ORA: Содержит конфигурацию службы прослушивания, которая принимает входящие соединения к базам данных.

3. Виды соединений:
   - Локальное соединение: Клиент и база данных работают на одном сервере.
   - Удаленное соединение: Клиент и база данные находятся на разных серверах, соединение осуществляется по сети.
   - Соединение через службу прослушивания: Подключение к базе данных через службу прослушивания, которая направляет запрос к нужному экземпляру базы данных.
   - Соединение напрямую: Подключение к базе данных напрямую, без использования службы прослушивания.

4. Строка подключения:
   - Строка, содержащая всю необходимую информацию для подключения к базе данных, включая сетевые параметры, имя базы данных, учетные данные пользователя и т.д.

5. Дескриптор подключения:
   - Структура данных, которая содержит информацию, необходимую для установления соединения с базой данных.

6. TNS - Transparent Network Substrate.
   - Это сетевая архитектура Oracle, обеспечивающая прозрачное сетевое взаимодействие между клиентами и базами данных.

7. Oracle Net Manager:
   - Графический инструмент для конфигурирования сетевых соединений Oracle Net.
   - Позволяет настраивать параметры подключения, службу прослушивания, а также другие сетевые компоненты.

8. Этапы запуска и останова экземпляра Oracle:
   - Запуск:
     1. Монтирование базы данных
     2. Открытие базы данных
   - Останов:
     1. Закрытие базы данных
     2. Размонтирование базы данных

9. Группа пользователей Windows, используемая Oracle для администраторов:
   - ORA_DBA - группа пользователей Windows, которая назначается администраторам Oracle Database.
